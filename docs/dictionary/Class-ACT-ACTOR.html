Description:

This is the `actor' class. The `actor' does it's message handling using the `behavior' function. There is asynchronous `tell' (no response) and synchronous `ask' and asynchronous `async-ask' (with response). To stop an actors message processing in order to cleanup resouces you should tell (either `tell' or `ask') the `:stop' message. It will respond with `:stopped' (in case of `[async-]ask').
